Bilibili 远古死号/三无“神号”全自动筛选工具

本工具是一个基于 Python 开发的通用 Bilibili UID 状态扫描器。它旨在协助用户批量检索并筛选出完全符合特定安全策略与昵称规则的远古三无账号（0级、0粉丝、0关注），适用于学术研究、个性化用户名盘点及远古存量账号行为分析。

🎯 核心特性

极致过滤算法：严格卡死 `0 级号`、`0 粉丝`、`0 关注` 三大核心指标，完美剔除所有活跃账号或高等级废弃账号。
高能昵称匹配：采用精确的正则表达式，联合判定昵称必须满足：**以小写字母开头**、**仅由小写字母和数字组成**、**总长度介于 3 至 18 个字符**。
柔性防封机制：内置阶梯式随机延迟（2.0 - 5.0 秒）以及 `-412`（请求过于频繁）智能拦截自适应冷却退避机制，保障脚本在单 IP 状态下能够长期稳定、无损地挂机运行。
全平台通用且无位数限制：全面解除对 UID 位数的硬编码限制，采用动态计算的控制台回擦机制，支持在 Windows、Linux（云服务器）及 macOS 上无缝部署。

🚀 环境准备

本工具基于 Python 3 编写，仅依赖官方的标准网络组件。

1. 安装 Python 环境
请确保您的系统中已安装 Python 3.6 或更高版本。

2. 部署依赖库
在您的终端（Terminal）或命令提示符（CMD）中执行以下命令安装 `requests` 库：
   bash
pip install requests

💻 快速开始

将代码文件 `bili_hunter.py` 下载至本地后，您可以通过以下两种方式启动程序：

方式一：交互引导模式（推荐）

在终端中直接运行脚本，程序将通过文本交互引导您输入检索范围与尾号限制：

   bash
python bili_hunter.py

输入引导示例：

开始 UID：输入您想要开始扫描的起始数字（例如：100000）
结束 UID：输入终止数字（例如：200000）
尾号后缀：如果您只想检索特定结尾的靓号（例如以 88 或 6 结尾），在此处输入对应的数字；若需全量扫描该区间，直接回车即可。

方式二：命令行位置参数模式（适合自动化脚本调用）

您也可以在启动命令后直接追加 `开始UID`、`结束UID`、`[可选尾号后缀]` 参数进行极速启动：

   bash
# 全量扫描 100000 到 100500 区间
python bili_hunter.py 100000 100500

# 仅扫描 100000 到 300000 区间内所有以数字 "6" 结尾的 UID
python bili_hunter.py 100000 300000 6


📂 结果输出与数据管理

1. 实时日志：程序运行期间，控制台将动态刷新当前的扫描进度、匹配百分比以及已捕获的有效账号数量。
2. 数据持久化：扫描结束后，工具会在当前脚本目录下自动生成或追加写入一个标准的 `.csv` 电子表格文件。
3. 文件名命名规范：文件名将自适应您的配置，如：`bili_scan_end88_100000_200000.csv`（其中 `end88` 代表指定了 88 尾号后缀，`all` 代表全量检索）。

该 CSV 文件支持使用 Excel、WPS 或各类文本编辑器直接打开，其输出的数据列结构如下：

| UID | 昵称 | 关注数 | 粉丝数 | 账号等级 |
| --- | --- | --- | --- | --- |
| 示例UID | 示例昵称 | 0 | 0 | Lv.0 |

⚠️ 免责声明与注意事项

1. 网络策略说明：脚本由于执行频率受限，单线程跑完大区间需要一定的时间（每 10,000 个全量 UID 约耗时 9.7 小时）。如需在大区间高并发检索，请自行修改代码并配合高匿代理 IP 池使用。
2. 全退避提示：由于 Bilibili 服务端风控机制的存在，如触发严重的 `-412` 拦截，脚本会自动触发 30-60 秒的随机休眠。若频繁触发该机制，建议暂停脚本，或重启路由器更换公网 IP 以恢复网络吞吐量。
